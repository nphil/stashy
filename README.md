# Stashy — iOS client for Stash

**A native iOS app for browsing, streaming, and downloading from your self-hosted
[Stash](https://github.com/stashapp/stash) media server.**

[![Platform](https://img.shields.io/badge/platform-iOS%2026%2B-blue)](https://www.apple.com/ios/)
[![Swift](https://img.shields.io/badge/Swift-6.0-orange)](https://swift.org)
[![UI](https://img.shields.io/badge/SwiftUI-Liquid%20Glass-lightgrey)](https://developer.apple.com/xcode/swiftui/)
[![Latest release](https://img.shields.io/github/v/release/nphil/stashy)](https://github.com/nphil/stashy/releases/latest)

> **This is the iOS version.** Stashy is a ground-up **native iOS 26 SwiftUI** app (built and
> daily-driven on iPhone) — not a web wrapper or a cross-platform port. An Android app is planned but
> has not started yet (see [Roadmap](docs/ROADMAP.md)).

Stashy connects to a Stash server you host yourself and focuses, in order, on: **fast playback and
scrubbing → direct-play first → on-device transcoding as the fallback → minimal load on your server →
privacy.**

---

## Features

### Playback
- **Direct play** of H.264 in MP4/MOV — instant start, hardware-decoded, instant seeks.
- **On-device remux** of HEVC and foreign containers (MKV/WebM) into a local loopback HLS stream —
  smooth playback with no server-side transcode.
- **Automatic server fallback** to Stash's HLS transcode only for formats Apple genuinely can't decode
  (e.g. 4:2:2/4:4:4 HEVC, VP9).
- **Manual quality/resolution override** from the player gear (forces Stash HLS at a chosen rung).
- **Playback speed** 0.25×–2× with pitch-corrected audio.
- **AI slow-motion** — on-device Neural-Engine frame interpolation (iOS 26 `VTFrameProcessor`) for true
  high-fps slow-mo instead of judder (opt-in beta).

### Scrubbing
- Frame-accurate seeking, variable-speed scrub, and **exact-frame preview** on downloaded files.
- Instant WebVTT sprite thumbnails, plus a **"most replayed" watch-heat overlay** built on-device.

### Downloads & offline
- **8-connection parallel download engine** — fast, resumable, and it keeps running when the app is
  backgrounded.
- **On-device transcode** to an iPhone-native MP4 (HEVC/H.264), including a checkpointed/resumable
  engine for long re-encodes.
- **Optional server-side transcode** via the [Stashy Companion plugin](stash-plugin/README.md): GPU
  HEVC (NVENC) or CPU AV1 (SVT-AV1) with **VMAF perceptual-quality targeting** — the smallest file that
  still looks good on a phone.
- Downloads and their sidecars live in **private, backup-excluded** storage; offline sprites + metadata
  power fully offline playback.

### Library & UX
- Scenes and performers browsing with native pull-down search and persisted sort.
- **Ratings** (0–5 stars) and **performer/tag favorites** with optimistic, instant updates.
- **Playability filter** (direct-play vs. needs-transcode), sourced from the companion plugin's report.
- iOS 26 **Liquid Glass** design, 120 Hz ProMotion, native gesture physics.

### Privacy
- **Face ID app lock** (immediate, no splash), **app-switcher snapshot blur**, and media-blur toggles.
- Diagnostics/telemetry are **off by default** and point at a configurable, self-hostable endpoint.

## Requirements

- An **iPhone on iOS 26 or later** (built and tested on iPhone 17 Pro).
- A reachable, self-hosted **[Stash](https://github.com/stashapp/stash) server** and its API key.
- A sideloading tool — **[AltStore](https://altstore.io) / SideStore / [Feather](https://github.com/khcrysalis/Feather)** — because the IPA is **unsigned** (this is not an App Store app).
- *(Optional)* the [Stashy Companion plugin](stash-plugin/README.md) on your Stash server for GPU/AV1
  server-side transcodes and codec reporting.

## Installation

Stashy ships as an **unsigned IPA** you sideload onto your own device.

- **Option A — add the app source** (gets updates): in AltStore/SideStore/Feather, add this repo's
  source manifest:
  ```
  https://raw.githubusercontent.com/nphil/stashy/main/apps.json
  ```
  then install **Stashy** from it.
- **Option B — direct download:** grab `Stashy.ipa` from the
  [latest release](https://github.com/nphil/stashy/releases/latest) and sideload it.

On first launch, enter your Stash server URL and API key (both stored in the iOS Keychain).

## Building from source

There is **no committed Xcode project** — it's generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen)
from [`ios/project.yml`](ios/project.yml), so new `.swift` files under `ios/Stashy/` are picked up
automatically.

```bash
brew install xcodegen
cd ios
xcodegen            # generates Stashy.xcodeproj
open Stashy.xcodeproj
```

- **Deployment target:** iOS 26.0 · **Language:** Swift 6 with `SWIFT_STRICT_CONCURRENCY: complete`.
- **Video engine:** FFmpeg comes from the Swift package
  [`nphil/stashy-videoengine`](https://github.com/nphil/stashy-videoengine) (LGPL-minimal:
  VideoToolbox H.264/HEVC + AAC encoders) — the app only links it.
- **CI is the canonical build.** GitHub Actions (`macos-15`) builds an unsigned IPA on every push to
  `main` and publishes a tagged Release with the IPA asset. Documentation-only and plugin-only pushes
  skip the build. See [`.github/workflows/ios-build.yml`](.github/workflows/ios-build.yml).
